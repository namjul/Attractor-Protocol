# Centers: Attractor Protocol Repository

The identified centers of this repository.
These are the focal points that organize everything else.
A gesture that weakens any of these without good reason should be questioned.

---

## Conceptual clarity

The protocol's ideas are accessible to someone encountering them for the first time.
Concepts build on each other in a readable order.
Jargon is either avoided or defined immediately when introduced.

**Healthy when:**
- A new reader can orient themselves from README.md without getting lost
- Terms are used consistently across all documents
- Examples are concrete enough to distinguish the concept from adjacent ones

**Degrading when:**
- New terms are introduced without definition
- The same concept appears under different names in different documents
- Examples are abstract or self-referential

---

## Epistemic discipline

The protocol practices what it preaches.
Claims about AP itself are falsifiable.
The schema enforces contact tests, not just structure.

**Healthy when:**
- Changes to the protocol include their own contact tests
- The schema templates resist vague or unfalsifiable answers
- Mythology in the protocol's own language gets flagged and revised

**Degrading when:**
- Claims about AP's value are made without evidence
- Template prompts can be answered with platitudes
- The protocol becomes a set of rules rather than a set of observations

---

## Usability

Someone can pick this up and use it without hand-holding.
The OpenSpec schema works in practice, not just in theory.
The effort to adopt is proportional to the value received.

**Healthy when:**
- A developer can install the schema and run a first gesture within one hour
- The templates feel like thinking aids, not compliance checklists
- The README answers the question "is this for me?" honestly

**Degrading when:**
- Adoption requires reading all six documents before starting
- Templates produce anxiety rather than clarity
- The gap between the methodology's ideals and daily practice is too wide to cross

---

## Contact test for this document

These centers are hypotheses, not facts.
If a center listed here turns out not to organize anything —
if removing it would change nothing about how the repo develops —
it should be dissolved or revised.

Review this document when opening a new gesture.
Update it when evidence suggests a center is stronger or weaker than described.
